## CONSENSUS {#consensus}

*Defined in `subworkflows/local/consensus/main.nf:15`*

**Keywords:** `consensus`, `variant calling`, `amplicon`, `alignment`, `trimming`, `SARS-CoV-2`, `monkeypox`

Generates a consensus genome from raw sequencing reads using a flexible amplicon-based workflow. Supports Illumina or Nanopore input, read cleaning, alignment, primer trimming, and consensus calling.

### Components

This workflow uses the following modules/subworkflows:

- `artic`
- `artic/filter`
- `bwa`
- `minimap2`
- `fastp`
- `seqyclean`
- `bbnorm`
- `ivar/consensus`
- `ivar/trim`
- `samtools/sort`
- `samtools/ampliconclip`
- `samtools/filter`
- `samtools/markdup`

### Inputs

| Name | Description |
|------|-------------|
| `ch_reads` | Paired-end Illumina reads Structure: [ val(meta), path(reads) ] |
| `ch_nanopore` | Nanopore reads Structure: [ val(meta), path(reads) ] |
| `ch_reference` | Reference genome FASTA Structure: [ path(fa) ] |
| `ch_primer_bed` | Primer BED file for amplicon trimming Structure: [ path(bed) ] |

### Outputs

| Name | Description |
|------|-------------|
| `consensus` | Consensus FASTA files generated by IVAR or ARTIC |
| `just_bam` | Aligned and sorted BAM files |
| `trim_bam` | Primer-trimmed BAM files |
| `clean_reads` | Quality- and adapter-trimmed reads |
| `filtered_reads` | BAM or fastq files after host/contaminant filtering |
| `for_multiqc` | Files for MultiQC input (e.g., QC reports from fastp or seqyclean) |
| `versions` | Files containing software versions Structure: [ path(versions.yml) ] |

**Authors:** Erin Young <eriny@utah.gov>


## INITIALIZE {#initialize}

*Defined in `subworkflows/local/initialize/main.nf:27`*

**Keywords:** `initialization`, `setup`, `metadata`, `input parsing`, `configuration`, `Cecret`

Initializes the Cecret pipeline by handling input files, verifying configurations, and  preparing references, primers, amplicons, and optional datasets. Also sets default values, checks for deprecated parameters, and generates structured input channels.

### Components

This workflow uses the following modules/subworkflows:

- `prep`
- `test`

### Inputs

| Name | Description |
|------|-------------|
| `sample_sheet` | Sample sheet with columns specifying sample names and read paths |
| `reads` | Directory containing paired-end Illumina reads |
| `single_reads` | Directory containing single-end Illumina reads |
| `fastas` | Directory of individual FASTA files |
| `multifastas` | Directory of multifasta files |
| `nanopore` | Directory of Nanopore reads |
| `reference_genome` | Reference genome FASTA file (optional if species is defined) |
| `primer_bed` | Primer BED file for trimming (optional if primer_set is provided) |
| `amplicon_bed` | Amplicon BED file for amplicon coverage analysis |
| `gff` | GFF file for variant annotation |
| `sra_accessions` | NCBI SRA accession(s) to fetch reads (Illumina only) |
| `genome_accessions` | NCBI genome accession(s) to fetch reference sequences |
| `primer_set` | Primer set name to use from included Cecret sets (e.g., mpx_yale) |
| `kraken2_db` | Path to Kraken2 database for host read filtering |
| `predownloaded_nextclade_dataset` | Nextclade dataset file if not downloading dynamically |

### Outputs

| Name | Description |
|------|-------------|
| `reads` | Combined Illumina reads [meta, reads] |
| `fastas` | Prepared FASTA sequences [meta, fasta] |
| `multifasta` | Multifasta file(s) for downstream processing |
| `nanopore` | Nanopore reads [meta, reads] |
| `reference` | Reference genome FASTA |
| `gff` | GFF annotation file |
| `primer` | Primer BED file used for trimming |
| `amplicon` | Amplicon BED file for ACI (Amplicon Coverage Inspection) |
| `versions` | Collected version metadata from submodules |
| `kraken2_db` | Kraken2 database path |
| `scripts` | Included legacy Python scripts for result processing |
| `nextclade_dataset` | Nextclade dataset file used for downstream analysis |

**Authors:** Erin Young <eriny@utah.gov>


## MSA {#msa}

*Defined in `subworkflows/local/msa/main.nf:7`*

**Keywords:** `multiple sequence alignment`, `phylogenetics`, `mafft`, `iqtree`, `snp-dists`, `phylogenetic tree`, `heatmap`, `visualization`

Multiple sequence alignment and phylogenetic analysis subworkflow.  Performs MSA using MAFFT, constructs a phylogenetic tree with IQ-TREE2,  computes pairwise SNP distances using snp-dists, and optionally visualizes  results using PhyTreeViz and HeatCluster.

### Components

This workflow uses the following modules/subworkflows:

- `mafft`
- `iqtree`
- `snpdists`
- `phytreeviz`
- `heatcluster`

### Inputs

| Name | Description |
|------|-------------|
| `ch_fasta` | Input FASTA file(s) for alignment |
| `ch_reference_genome` | Reference genome in FASTA format (optional, used in MAFFT) |

### Outputs

| Name | Description |
|------|-------------|
| `tree` | Newick tree file produced by IQ-TREE2 |
| `matrix` | SNP distance matrix file from snp-dists |
| `msa` | Multiple sequence alignment file produced by MAFFT |
| `for_multiqc` | Visualization outputs for MultiQC integration from PhyTreeViz and/or HeatCluster |
| `versions` | Collected tool version information |

**Authors:** Erin Young <eriny@utah.gov>

## QC {#qc}

*Defined in `subworkflows/local/qc/main.nf:14`*

**Keywords:** `quality control`, `fastqc`, `kraken2`, `samtools`, `amplicon`, `variant calling`, `bcftools`, `igv`, `ivar`, `visualization`, `multiqc`

Comprehensive quality control and variant reporting subworkflow integrating multiple tools  such as FastQC, Kraken2, Samtools, ACI, iVar, BCFtools, and IGV Reports.  Generates outputs for summary statistics, variant calling, coverage, and MultiQC visualization.

### Components

This workflow uses the following modules/subworkflows:

- `fastqc`
- `kraken2`
- `kraken2/db`
- `samtools/ampliconstats`
- `samtools/plot/ampliconstats`
- `samtools/qc`
- `aci`
- `bcftools`
- `ivar/variants`
- `igv/reports`

### Inputs

| Name | Description |
|------|-------------|
| `ch_raw_reads` | Raw reads input as [meta, reads] |
| `ch_clean_reads` | Cleaned reads input as [meta, reads] |
| `ch_kraken2_db` | Path to Kraken2 database |
| `ch_initial_bam` | Initial BAM input as [meta, sam] |
| `ch_trim_bam` | Trimmed BAM and BAI input as [meta, bam, bai] |
| `ch_reference_genome` | Reference genome in FASTA format |
| `ch_gff_file` | GFF annotation file (used with iVar) |
| `ch_amplicon_bed` | BED file of expected amplicons (for ACI) |
| `ch_primer_bed` | BED file of primer coordinates (for ampliconstats) |

### Outputs

| Name | Description |
|------|-------------|
| `for_multiqc` | MultiQC-compatible outputs from FastQC, Kraken2, Samtools, and ACI |
| `for_summary` | Summary files including read QC, variant calls, coverage stats, and amplicon analysis |
| `versions` | Tool version information from all modules used |

**Authors:** Erin Young <eriny@utah.gov>

## TEST {#test}

*Defined in `subworkflows/local/test/main.nf:4`*

**Keywords:** `ena`, `sra`, `ncbi datasets`, `read download`, `reference genomes`, `data ingestion`

Subworkflow to download sequencing reads from ENA and reference genomes using the NCBI datasets tool. Supports both SRA accession-based read download and genome accession-based reference download.

### Components

This workflow uses the following modules/subworkflows:

- `ena`
- `datasets`

### Inputs

| Name | Description |
|------|-------------|
| `ch_sra_accessions` | List of SRA accession strings to fetch FASTQ files from ENA |
| `ch_genome_accessions` | List of genome accession identifiers to fetch reference sequences using NCBI datasets |

### Outputs

| Name | Description |
|------|-------------|
| `reads` | Channel of [meta, reads] tuples from ENA downloads (paired or single-end) |
| `fastas` | Fasta files for genomes downloaded using NCBI datasets |
| `versions` | Version metadata for ENA and datasets tools used |
